# consensflow-site

The marketing landing page for **ConsensFlow** — consult one named AI participant at a time, with a
handoff of your current session, from inside Pi or Claude Code.

Sibling of [kluris-site](https://kluris.ngvoicu.dev) and [specmint-site](https://specmint.ngvoicu.dev); same
zero-build, single-file static stack.

## Stack

- **One self-contained `index.html`** — HTML + inline CSS + inline vanilla JS. No framework, no build
  step, no `node_modules`.
- Fonts: DM Serif Display (headings), DM Sans (body), JetBrains Mono (code) — via Google Fonts.
- Brand: cyan→indigo "flow" gradient; per-engine accent colors (Claude Code, Codex, Pi, OpenCode,
  image) reused across the routing diagram and the preset gallery.

## Local dev

```bash
python3 -m http.server 4500   # then open http://localhost:4500
```

## Data: the preset gallery

The "50 ready-made participants" gallery is rendered from data **generated from ConsensFlow's source
of truth** (`consensflow-cc/lib/presets.js`) — never hand-transcribed. To regenerate after the preset
catalog changes:

```bash
node --input-type=module -e '
import { PARTICIPANT_PRESETS } from "../consensflow-cc/lib/presets.js";
import { writeFileSync } from "node:fs";
const slim = PARTICIPANT_PRESETS.map(p => ({preset:p.preset,id:p.id,name:p.name,label:p.label,kind:p.kind,model:p.model,effort:p.effort??null,thinking:p.thinking??null,description:p.description}));
writeFileSync("assets/presets.json", JSON.stringify(slim,null,2));
'
```

then inline the JSON into `index.html` in place of the `<script id="cf-presets">` body.

## Assets

- `assets/mark.png`, `assets/og-image.jpg` — the ConsensFlow brand mark (converging streams → one
  node), **generated by ConsensFlow's own `@pygmalion` image participant** (gpt-image-2). Source at
  `assets/logo-pygmalion.png`.
- `assets/favicon.svg` — inline vector mark (crisp at small sizes).
