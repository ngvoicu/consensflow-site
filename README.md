# consensflow-site

The marketing landing page for **ConsensFlow** тАФ consult one named AI participant at a time, with a
handoff of your current session, from inside Pi or Claude Code.

Sibling of [kluris-site](https://kluris.ngvoicu.dev) and [specmint-site](https://specmint.ngvoicu.dev); same
zero-build, single-file static stack.

## Stack

- **One self-contained `index.html`** тАФ HTML + inline CSS + inline vanilla JS. No framework, no build
  step, no `node_modules`.
- Fonts: DM Serif Display (headings), DM Sans (body), JetBrains Mono (code) тАФ via Google Fonts.
- Brand: cyanтЖТindigo "flow" gradient; per-engine accent colors (Claude Code, Codex, Pi, OpenCode,
  image) reused across the routing diagram and the preset gallery.

## Local dev

```bash
python3 -m http.server 4500   # then open http://localhost:4500
```

## Data: the preset gallery

The "50 ready-made participants" gallery is rendered from data **generated from ConsensFlow's source
of truth** (`consensflow-cc/lib/presets.js`) тАФ never hand-transcribed. To regenerate after the preset
catalog changes:

```bash
node --input-type=module -e '
import { PARTICIPANT_PRESETS } from "../consensflow-cc/lib/presets.js";
import { writeFileSync } from "node:fs";
const slim = PARTICIPANT_PRESETS.map(p => ({preset:p.preset,id:p.id,name:p.name,label:p.label,kind:p.kind,model:p.model,effort:p.effort??null,thinking:p.thinking??null,description:p.description}));
writeFileSync("assets/presets.json", JSON.stringify(slim,null,2));
'
```

then inline the JSON into `index.html` in place of the `<script id="cf-presets">` body.

## Deploy

GitHub Pages from the repo root, like the sibling sites. Add a `CNAME` file for a custom domain when
one is chosen (left unset for now).

## Deploying (read this тАФ Pages builds get stuck)

Push `main` to `upstream` (GitHub) and Pages builds automatically тАФ but builds **regularly hang in `"building"`**. After every push:

```bash
gh api repos/ngvoicu/consensflow-site/pages/builds/latest --jq .status   # expect "built" within ~1 min
gh api -X POST repos/ngvoicu/consensflow-site/pages/builds               # re-queue if it sits in "building"
```

Then confirm the change is actually live: `curl -s https://consensflow.ngvoicu.dev/ | grep '<something-from-your-change>'`.
